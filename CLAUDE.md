# sakura-homonkiroku — 訪問指導記録票アプリ

## データソースについて

企業・実習生データは **PIIA SAKURAセッションが自動管理する**。
このセッションでデータを手動編集・追加する必要はない。

| ファイル | 説明 | 編集可否 |
|---------|------|---------|
| `index.html` 内 `let DB = {...}` | 実行時DBブロック | **自動生成・編集禁止** |
| `db.json` | DBのミラー | **自動生成・編集禁止** |
| `gas/Code.gs` | GAS（PDF生成・メール・カレンダー） | 編集可 |

## このセッションの作業範囲

このセッションで対応するのは以下に限る:

1. **GASスクリプトの修正** (`gas/Code.gs`) — PDF レイアウト、メール文面、カレンダー登録ロジック
2. **UIの改修** (`index.html` のスタイル・フォーム構造） — ただし `let DB = {...}` ブロックには触れないこと
3. **担当者リスト変更** — `staff:[]` の内容のみ。変更が必要な場合はユーザーに確認してから編集する

## データ更新の仕組み（参照のみ）

PIIA SAKURAセッションで以下のファイルが更新されると、フックにより自動的に同期される:

- `../PIIA SAKURA/data/companies.json`
- `../PIIA SAKURA/data/trainees.json`

同期スクリプト: `../PIIA SAKURA/tools/sync_homonkiroku.py`
